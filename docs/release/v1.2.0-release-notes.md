# Voice v1.2.0 — Release Notes

## Major Fixes

### Authentication Fixed
- **Login/Signup now works**: After successful login or signup, the modal closes and returns to the app. Previously users were trapped in the auth modal.
- **Password reset**: Users can now reset their password from the login screen via email.
- **Account deletion**: Users can delete their account from the Profile screen (Play Store compliance).
- **Friendly error messages**: Auth errors now show human-readable messages instead of raw Firebase error codes.
- **Email validation**: Client-side email format validation before sending to Firebase.
- **Keyboard handling**: Login/Signup screens now handle keyboard avoidance properly.

## Major Features

### Favourites / Pinned Phrases
- Users can star any sentence in the AAC Board to save it as a favourite.
- Favourites panel accessible from the sentence bar (star icon).
- Tap a favourite to instantly load and speak it.
- Stored locally in AsyncStorage — works offline, survives restarts.
- Maximum 50 favourites.

### Persistent Sentence History
- Sentence history now survives app restarts (previously in-memory only).
- Shows speak count for frequently repeated sentences.
- Duplicates are automatically deduplicated (moved to top).
- Maximum 100 entries stored locally.

### Vertex AI Context-Aware Phrase Suggestions
- New Cloud Function (`aacPhraseSuggestions`) uses Vertex AI Gemini to generate context-aware AAC phrase completions.
- Considers: current sentence, recently spoken phrases, time of day.
- AAC-specific system prompt ensures suggestions are short, simple, child-friendly, and never contain medical advice.
- Suggestions appear in the AI suggestions strip alongside local bigram predictions.
- Fully optional — local predictions work without internet.

### Image-to-AAC Phrases (Vertex AI Vision)
- New Cloud Function (`imageToAACPhrases`) uses Vertex AI Gemini Vision.
- When a user takes or selects a photo, the app now shows tappable AAC phrase chips describing the scene.
- Example: photo of a dog → "I see a dog", "it is big", "I like it", "can I touch"
- Phrases are communication-ready, not just captions.

## GCP Integration

### Cloud Functions (3 endpoints)
1. `imageCaptionProxy` — Hugging Face image captioning (existing, unchanged)
2. `aacPhraseSuggestions` — Vertex AI Gemini context-aware phrase suggestions (NEW)
3. `imageToAACPhrases` — Vertex AI Gemini Vision image-to-AAC phrases (NEW)

### Setup Required
```bash
# Set Vertex AI project (already matches Firebase project)
firebase functions:config:set vertex.project_id="commai-b98fe"

# Ensure Cloud Functions service account has Vertex AI User role:
# GCP Console → IAM → find the service account → Add Role → Vertex AI User

# Deploy
firebase deploy --only functions
```

### Architecture
- All AI secrets stay server-side in Cloud Functions
- Client calls plain HTTPS endpoints — no API keys in the mobile app
- Uses Application Default Credentials (ADC) for Vertex AI — no separate key management
- Graceful fallback: if Vertex AI is unavailable, local predictions still work

## Auth Architecture

```
User taps Login/Signup
→ Modal opens (LoginScreen or SignupScreen)
→ Client-side validation (email format, password length)
→ Firebase Auth call (signInWithEmailAndPassword / createUserWithEmailAndPassword)
→ On success: navigation.goBack() closes modal
→ AuthContext.onAuthStateChanged fires → user state updates globally
→ SettingsContext detects auth → syncs settings from Firebase

Password Reset:
→ sendPasswordResetEmail(auth, email)
→ Firebase sends reset email
→ User resets via email link

Account Deletion:
→ ProfileScreen → Delete Account
→ Confirmation dialog
→ Removes user data from Realtime Database
→ Calls deleteUser(currentUser)
→ Auth state clears
```
